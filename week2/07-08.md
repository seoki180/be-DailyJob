###### 1. 오늘 완료한 것
- 오늘 실제로 끝낸 기능/수정/학습:
  - 김영한 강의를 통해 서블릿 구조 파악 → WAS 기반 구조로 전환 결정
  - 기존 REST 방식에서 SSR(Server-Side Rendering) 방식으로 전면 전환
  - index.html 등 정적 파일 방식에서 Page Router 방식으로 구현 완료
- (오늘의 목표 중에서) 아직 끝내지 못한 부분:
  - Request Handler가 비대해진 문제 → Controller로 분리 예정 (미완)

###### 2. 오늘 가장 의미있었던 기술적 판단
- 내가 내린 판단: REST API 방식을 버리고 SSR + Page Router 구조로 전환
- 그렇게 판단한 이유: 김영한 강의에서 서블릿 구조를 파악한 후, WAS를 활용하는 방향으로 설계를 재검토했고 SSR이 현재 구조에 더 적합하다고 판단
- 고려했지만 선택하지 않은 대안: 기존 REST + index.html 정적 파일 방식 유지
- 이 판단이 코드 품질/테스트/유지보수에 미친 영향: Page Router 도입으로 라우팅 구조가 명확해졌으나, Request Handler에 로직이 집중되어 Controller 분리가 필요한 상황 발생
